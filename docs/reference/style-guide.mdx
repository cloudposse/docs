## Markdown

### Headings

```
# Heading 1

Note that these headings are not shown in the table of contents.
## Heading 2.a
### Heading 3
#### Heading 4
Note that headings more than 3 deep are not shown in the table of contents
## Heading 2.b
```

### Lists

#### Unordered Lists

Useful for arranging items without a specific order.

```

```

### Markdown Admonitions

```
:::warning Tip title

This is a tip.

:::
```

:::note Markdown admonitions are sensitive to newlines

Make sure you put a newline after the opening `:::` and before the closing `:::`.

:::

**NOTE:** While this is the preferred way of doing admonitions, some components will not render markdown. If that happens, use the [Admonition component below.](#Admonition)

```mdx
<Admonition type="tip">
  This is a tip.
</Admonition>
```

<Admonition type="tip">
  This is a tip.
</Admonition>

### Code blocks

Surround code with three backticks to create a code block. Include the language after the first set of backticks to enable syntax highlighting.

```bash title="Example title"
echo "Hello, World!"
```

## Conformance and Best Practices

- Do not use `<br />` for spacing. Instead, update the CSS
- Prefer to use markdown admonitions with `:::` instead of the `<Admonition />` tag. Only use `<Admonition />` when markdown is not rendering correctly.
- Definitions should use `<dl />` and `<dt />` instead of headings
